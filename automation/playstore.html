How to Use This Tool
1. Launch App on Phones

Type your app's package name (like com.yourname.app) and tap Launch. The app opens on all phones at once or one by one if you pick a delay. You can find your package name in Android Studio or in your build.gradle file.

2. Type on Phones

Sends keyboard input to the phone. Use "Send to: All phones" to type the same thing everywhere, or pick a specific phone to type only on that one. The buttons (Enter, Back, Home, etc.) send a single key press.

3. Quick Actions

One-tap buttons for common things: take a screenshot, swipe the screen in any direction, lock/wake the phone, go home, clear recent apps. Screenshot files are saved to /sdcard/ on each phone.

4. Test Loop

Phones automatically switch between a website and your app on a timer. This lets you test both at the same time — useful for checking how your app handles being interrupted by the browser, and for ad impression testing on your website.

RAM Warning

Each AVD emulator uses about 1.5 GB of RAM. The green bar at the top shows how much of your computer's memory is in use. If it turns yellow or red, close some phones from the main Dashboard before running tests — otherwise your computer will slow to a crawl.
